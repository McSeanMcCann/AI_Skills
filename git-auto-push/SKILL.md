# Git Auto-Push — ALWAYS ON (Global, All Projects)

**This is a non-negotiable global policy that applies to every workspace, every session, without exception.**

After every file change, run the following sequence in the terminal:

## Workflow

1. **Check for git repository** — If the workspace does not contain a `.git` folder, ask the user: "This workspace is not a git repository yet. Please provide the GitHub repo URL to initialize it." Then initialize git and set the remote.

2. **Get the repo URL** — If not already set as the remote, ask the user: "What is the GitHub repo URL for this project?"

3. **Initialize if needed** — If the folder is not a git repo:
   ```
   git init
   git remote add origin <repo-url>
   ```

4. **After each file edit**, immediately run:
   ```
   git add <changed-file(s)>
   git commit -m "<auto-generated message>"
   git push origin main
   ```

4. **Commit message format** — Generate from the actual changes:
   - `Add <filename>` for new files
   - `Update <filename>: <brief description of what changed>`
   - `Delete <filename>` for removals
   - For multi-file changes: `Update <n> files: <short summary>`

5. **If push fails** (no upstream set):
   ```
   git push --set-upstream origin main
   ```

## Notes

- Use `main` as the default branch. If the repo uses `master`, adapt accordingly.
- If the remote already exists, skip `git remote add`.
- Always run `git push` after `git commit` — do not batch commits.
- Do not wait for the user to ask. Push automatically after every change.
- Each workspace defines its own remote URL — do not use a hardcoded URL from another project.
- The per-project remote is set in that project's `copilot-instructions.md` or via `git remote -v`.
